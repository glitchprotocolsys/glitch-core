# GLITCH PROTOCOL - DEPLOYMENT COMMAND CENTER MANIFEST

## 1. Core Architectural Framework
The protocol operates on a strict transactional friction model designed to enforce continuous deflation while building an insulated asset floor:
* **0.80% Core Furnace Burn:** Captured from every transaction block and routed directly to an isolated burning hub.
* **0.20% Sovereign Treasury Routing:** Re-routed into a separate pool to accumulate native L1 SOL yield.

## 2. Active Station Codebase

### A. Background Data Hub (`streamer.py`)
Manages price action loops, simulated community growth trends, ledger logs, and threat anomalies (whale transactions and fast-block sniper attempts).

### B. Visual Console Matrix (`dashboard.js`)
A responsive 6-panel terminal interface built using `blessed-contrib` that handles real-time visual updates and features keyboard hotkey layers:
* `[S]` Switches directly to the Predictive Economic Simulation Overlay.
* `[M]` Snaps back immediately to the active transaction ledger stream.
* `[Q] / [ESC]` Kills the console processes and exits back to root prompt.
